## 遮擋判定方法

第一版可以定義成：

> 結合 **visibility-based filtering** 與 **velocity-based outlier detection** 的關節可靠性判定方法。

也就是不直接判斷「是哪一種遮擋」，而是判斷：

> 這個關節點在這一幀是否可靠，能不能拿來算 RULA。

---

## 1. Visibility-based filtering

使用 pose model 輸出的 visibility 判斷關節點可信度。

建議設定：

```python
VIS_TH = 0.5
VIS_MID_TH = 0.8
```

判斷邏輯：

```python
if visibility < 0.5:
    unreliable = True
```

意思是：

> 當 visibility 低於 0.5，直接視為低可信度關節點。

但要注意：

> visibility 高不代表一定正確，所以不能只靠 visibility。

---

## 2. Velocity-based outlier detection

計算關節點在時間序列中的速度異常。

公式：

```python
dt = frame_interval / fps
jump = distance(curr_joint, prev_reliable_joint)
speed_ratio = (jump / dt) / body_scale_ref
```

其中：

```text
dt：兩個分析幀之間的實際時間差
jump：目前關節點與上一個可靠關節點的距離
body_scale_ref：人體尺度參考值，例如肩寬
```

這樣做的目的是把關節位移轉成：

> 每秒移動了多少個人體尺度。

---

## 3. 兩者搭配方式

不建議用權重占比，第一版用規則式搭配即可：

```python
if visibility < VIS_TH:
    unreliable = True
    reason = "low_visibility"

elif speed_ratio > SPEED_TH_HIGH:
    unreliable = True
    reason = "extreme_velocity_anomaly"

elif visibility < VIS_MID_TH and speed_ratio > SPEED_TH_LOW:
    unreliable = True
    reason = "medium_visibility_with_velocity_anomaly"

else:
    unreliable = False
    reason = "reliable"
```

意思是：

```text
visibility 很低 → 直接判不可靠
速度極端異常 → 即使 visibility 高，也判不可靠
visibility 中等偏低 + 速度異常 → 判不可靠
其他情況 → 暫時視為可靠
```

## 實驗設計

### 目的

建立正常情況下的 `speed_ratio` 分布，決定速度異常門檻：

```python
SPEED_TH_LOW
SPEED_TH_HIGH
```

---

### 拍攝資料

第一版只拍：

```text
正面、無遮擋、骨架穩定的影片
```

不要先用背面、45 度或側面影片統計 threshold，因為那些角度容易自遮擋，會污染正常速度分布。

---

### 動作內容

拍三種即可：

```text
1. 站立不動 10 秒
2. 慢速伸手、收手 20 秒
3. 正常速度重複拿取物品 30 秒
```

這三種分別用來涵蓋：

```text
模型本身抖動
慢速正常動作
RULA 作業型正常動作
```

---

### 分析流程

1. 用你的系統跑骨架偵測。
2. 設定不同 frame interval，例如：

```text
frame_interval = 1, 5, 10
```

3. 對每個 RULA 相關關節計算：

```python
speed_ratio = (jump / dt) / body_scale_ref
```

4. 依部位分組統計：

```text
trunk：shoulder、hip
head：nose、ear
arm：elbow
hand：wrist、index、pinky
```

5. 用百分位數決定 threshold：

```python
SPEED_TH_LOW = 95th percentile
SPEED_TH_HIGH = 99th percentile
```

---

## 測試遮擋判定效果

threshold 不要用遮擋影片來定，而是用遮擋影片來測試。

測試資料可以包含：

```text
正面遮擋
45 度拍攝
側面拍攝
背面拍攝
手被桌子或工具遮擋
手與身體重疊造成自遮擋
```

觀察系統是否能把明顯錯誤或不穩定的關節標成 unreliable。

---

## 論文寫法

可以寫成：

> 本研究結合關節點 visibility 與時間序列速度一致性進行關節可靠性判定。首先，當關節點 visibility 低於 0.5 時，直接視為不可靠。其次，計算相鄰分析幀之關節位移，並根據實際時間差與人體尺度進行正規化，得到 normalized joint velocity。當 normalized velocity 超過正常可見影片中統計所得之門檻時，該關節點被視為速度異常。透過此方法，可偵測低可見度關節點以及 visibility 較高但位置明顯跳動之錯誤預測。
